## 5. 结论

1. 批量删除在三个引擎均显著提升，提升区间为 4.13x 到 16.36x。
2. 批量克隆在 Redis 与 TiKV 上提升明显，分别为 4.20x 与 2.06x。
3. MySQL 批量克隆在本轮统一逻辑测试中出现明显回退（0.28x，约 -72.4%），需要后续专项分析。

## 6. 后续建议

1. 对 MySQL clone 单独做 profile 与 SQL 慢日志分析，定位瓶颈点。
2. 对 MySQL clone 增加分批大小与并发参数扫描，验证最优配置区间。
3. 复测 2 到 3 轮 MySQL clone，确认结果是否稳定可复现。
